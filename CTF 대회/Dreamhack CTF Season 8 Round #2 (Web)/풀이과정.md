다른 부분은 크게 볼만한 게 없고 여윽시 app.py만 좀 보면 되는데

플래그를 얻으려면 /admin 으로 접속하면 볼 수 있는데 /admin 은 로그인만 성공하면 자동으로 접속이됨

근데 살짝 신기한 점은 로그인할 때 따로 guest, user, admin 구분 없이 로그인만 되면 된다는 거임

실제로 users 데이터베이스에는 admin밖에 없기도 하고, 그래서 로그인을 하면 되는데 당연히 유저이름은 알아도 비밀번호는 모름


sql문을 쓰는 부분을 보면 /login과 /search가 있는데 /login에서

`cursor.execute("SELECT * FROM users WHERE username = ? AND password = ?", (username, password))`

가 있는데 여기서 username과 password를 인자로 따로 넘겨줌. 이렇게 되면 이 부분을 서버에서는 단순한 값으로 치부해버려서 sql 인젝션을 해도 단순한 데이터로 치부해버리기 때문에 서버는 '특이하게 생긴 문자열이네~' 하고 처리함.

그렇기 때문에 /login sql인젝션을 할 수 없어서 /search에서 해볼 수 있는지 확인하면 되는데

/search를 보면 앞에서 select를 하기만 하고 따로 출력하는 부분은 없음

select의 결과값을 통해 조건문으로 분기만 해서 select 결과값으로 할 수 있는 게 없음

그런데 이번에 **Blind sql Injection** 이라는 게 있다는 걸 알게됨 blind sql injection은 결과로 출력되지 않더라도 인젝션의 결과에 따라 생기는 변화, 이 문제에서는 /search에서 Found! 또는 Not Found! 를 보고 sql이 되고있는지 확인할 수 있다는 건데

/search에서 `%'; UPDATE users SET password = 'admin' WHERE username = 'admin';--` 이렇게 입력을 하면 코드가 2개로 나뉘어서 

`SELECT COUNT(*) FROM secret_data WHERE title LIKE '%%';`
`UPDATE users SET password = 'admin' WHERE username = 'admin';--%'`

이렇게 되는데 이건 안 됨 왜냐하면 sqlite3에서는 한 번에 하나의 sql 문장만 실행하도록 되어 있어서 위 처럼 2개의 문장을 겹쳐놓은 건 안 됨


그러면 어카지 하다가 비밀번호를 굳이 찾는 게 아니라 비밀번호를 UPDATE로 바꾸거나 INSERT로 삽입하는 것도 생각해봤는데 그것도 안 된데 INSERT나 UPDATE는 반환값이 없어서 WHERE이나 UNION을 쓸 수 없음

그렇기 때문에 select를 한번 더 써서 password의 값을 한 자리씩 유추해나가는 방식(Blind SQL Injection)을 할 수 밖에 없음

근데 hex값에 16자리라서 손으로 일일히 하는 건 너무 빡세고 하는 파이썬 코드를 짰음

%' AND (SELECT SUBSTR(password,{i},1) FROM users) = '{char}'--
