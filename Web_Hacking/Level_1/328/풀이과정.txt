보안을 강화하려면 strcmp(..., ...) === 0 과 같이 =를 3개를 쓰거나 PHP 8.0 이상을 쓰면 strcmp에서 배열을 넣으면 중단되어서 안전함

근데 문제가 내가 POST 요청을 보내는 방법을 몰랐네? ㅅㅂ

난 단순히 GET은 페이지를 보여달라는 것만 알고 있었고, POST는 데이터를 담아서 보내는 거라고 생각했었는데 그게 아니라 GET도 데이터를 담을 수 있었음

URL에서 (웹페이지 주소)/?(GET 요청 데이터) 와 같이 '?' 뒤에 오는 것은 다 GET요청의 데이터임 

POST요청을 보내고 싶으면 Post Man같은 도구를 따로 쓰거나 따로 html 코드를 작성해서 거기서 post를 보낼 수도 있음

html은 문제 폴더에 따로 temp.html 작성해뒀으니까 참고

cmd 에서는 curl -X POST "http://host3.dreamhack.games:13374/" -d "password[]=anything"

과 같이 작성하면 curl을 통해서 POST를 보낼 수 있음



중요하게 알고 넘어가야 할 점 
- GET은 단순히 페이지를 달라고 요청만 하는 게 아니라 **URL**에 데이터를 담아서 보낼 수 있음 (ex. http: //example.com/page?이름1=값1&이름2=값2)
- GET은 URL에 데이터를 담지만 POST는 HTTP Body에 데이터를 담음
- POST 요청을 보내기 위해서는 URL에 값을 담아서는 안 되고 Postman 같은 도구나 cmd에서의 curl 명령어 또는 html 코드를 따로 작성해서 post를 보내거나 해야함
