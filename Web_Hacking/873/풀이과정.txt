어려운 문제는 아니었지만 그렇다고 해서 막 그렇게 쉽게 풀지도 못한 문제

가상서버를 열어서 웹페이지로 들어갔을 때 닉네임과 비밀번호를 입력하는 창이 나옴
여기서 알맞은 값을 입력하고 Step2로 나아가야함

닉네임과 비밀번호는 받은 문제파일의 step2.php 에서 

...
if (preg_match("/[a-zA-Z]/", $input_pw)) {
    echo "alphabet in the pw :(";
}
else{
    $name = preg_replace("/nyang/i", "", $input_name);
    $pw = preg_replace("/\d*\@\d{2,3}(31)+[^0-8\"]\!/", "d4y0r50ng", $input_pw);
...

와 같은 부분이 있는데 각 부분을 해석하자면
처음에 pw에 알파벳이 들어있다면 탈락
name에서 'nyang'이 들어있다면 빈칸으로 치환
pw에서 정규식을 만족하는 pw가 들어오면 d4y0r50ng로 치환함

아래는 위를 만족하는 name과 pw
name = dnyanyangng0310
pw = 0@000319!+1+13


Step2에 들어가면 cmd를 입력할 수 있는 입력칸이 하나 있음 여기에 알맞은 입력값을 넣으면 flag값을 볼 수 있음
근데 여기서 cmd에 필터링이 들어가는데 필터링 내용이 preg_match("/flag/i", $cmd) 으로 "/flag"가 들어있으면 탈락!
그래서 입력값에 "/flag"이 있으면 안 됨. 

리눅스에서 \는 특수문자를 일반문자 취급하는 기능이 있는데 \뒤에 일반문자가 오면 그냥 일반문자로 취급해서
/fl\ag => /flag 로 인식함 따라서 ../dream/fl\ag.tx 와 같이 입력하면 "/flag"가 아니라서 필터를 통과하고 플래그를 얻을 수 있음
